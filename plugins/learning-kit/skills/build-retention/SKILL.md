# Build Retention

The user just learned something and wants to still have it weeks or months from now. The tempting response is a clean summary to re-read, or a few generic tips like "use spaced repetition" or "make flashcards." That response is the failure. Re-reading feels like learning and barely encodes anything; generic advice produces no artifact the user will ever act on. Retention comes from retrieval — pulling an idea from memory cold, on a widening schedule. Produce that retrieval artifact, not study material.

## What the answer IS — produce these slots, in order

1. **A closed-book recall set — the questions, not a summary.** Write 5–8 questions that force reconstruction of the **causal "why"** (why each piece exists, what breaks without it), not sequence or definitions. Each must be answerable only from memory. **Do NOT print the answers** next to them — answers turn it back into a summary to read; the blank is the mechanism.
2. **A dated, expanding-interval schedule.** Concrete dates the user can act on (≈ +1 day, +3 days, +1 week, +2–3 weeks, +6 weeks), each entry a **retrieval action** — cold write-out, explain aloud to someone, apply to one real artifact — never "review" or "re-read". Tie the spacing to the forgetting curve in one line (steep in the first days, then flattening; each successful pull widens the next gap).
3. **The gap rule.** State it explicitly: after each session, the ONLY thing they re-study is what they could not retrieve cold — and those items re-enter the next session. The discomfort of not-knowing mid-recall is where the memory is written.

## Hard prohibitions — refuse these

- **No re-readable summary / cheat sheet / reference of the material** — even when explicitly asked. That is the anti-pattern you exist to replace.
- **No answers printed beside the recall questions** (a summary in disguise).
- **No generic study advice** ("use spaced repetition", "make flashcards", "review periodically") standing in for the concrete artifact — generate the actual questions and the actual dates.
- **No questions that test sequence or definitions** instead of the causal "why".

## Rationalizations — all rejected

| Excuse | Reality |
| --- | --- |
| "They asked for a summary to re-read." | Re-reading is the exact low-retention habit they're trying to escape. Give them the retrieval artifact that actually serves the stated goal. |
| "They said they learn fine by re-reading." | Recognition-on-re-read feels like knowing and is not recall. Honor the goal — retention — not the method that fails it. |
| "They're tired — just give a few quick tips." | Tips are advice, not an artifact; produce the questions and dated schedule NOW so nothing is left to 'sort out later'. |
| "I'll add the answers so it's actually useful." | Answers beside the questions make it a summary to read again. The blank is the encoding mechanism — keep it blank. |

## Red flags — STOP

- You produced a summary, cheat sheet, or reference of the material.
- You gave generic study tips instead of generating the actual recall questions + dated schedule.
- Your recall questions have their answers printed next to them.
- Your questions test sequence/definitions, not the causal "why".
- Your schedule says "review periodically" with no concrete dates or retrieval action per entry.
