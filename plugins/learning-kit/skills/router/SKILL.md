# Learning Router

Six learning skills exist, all user-invoked — so nothing fires them automatically and you have to remember which fits the moment. This is the index. Tell me where you are with the material; I name the **one** skill to invoke and why, then hand off. I do not teach here — I route.

## Route by where you are with the material

| You are… | Invoke | Because |
| --- | --- | --- |
| **drowning in jargon — it won't click** (you do NOT understand it yet) | `confusion-translator` | finds the one keystone idea and translates it into plain life, gating on comprehension |
| **fresh off learning it — want to know if it actually stuck** | `forced-feynman` | you explain it to a 10-year-old; every gap gets stopped at, you repair it |
| **sure you've mastered it — want that disproven** | `hidden-gaps-detector` | five deceptively simple questions + a blunt verdict on what your foundation is missing |
| **wanting to learn a concept by doing and failing, not by being told** | `real-mistakes-simulator` | drops you in a realistic scenario, withholds the answer when you err |
| **needing to be functional FAST** (hours, not a syllabus) | `curve-destroyer` | the highest-leverage 20%, what to ignore, one keystone rep |
| **holding a concrete outcome + a deadline** | `plan-architect` | a day-by-day plan, one 45-minute task per day, traced back to the outcome |

## Disambiguate the two close pairs

- **`confusion-translator` vs `forced-feynman`** — split by direction. You can't follow it yet → `confusion-translator` (input: it gets translated *to* you). You think you can already explain it → `forced-feynman` (output: you explain it and get stopped at the gaps).
- **`hidden-gaps-detector` vs `real-mistakes-simulator`** — split by intent. You want your *claimed mastery* falsified → `hidden-gaps-detector` (it interrogates, blunt verdict). You want to *build* the skill through struggle → `real-mistakes-simulator` (it runs an error loop until automatic).
- **`curve-destroyer` vs `plan-architect`** — split by time horizon. One sitting, get usable now → `curve-destroyer`. Multi-day, aimed at a checkable outcome by a date → `plan-architect`.

## How to hand off

State the matched skill by name and the one-line reason, then tell the user to invoke it (these skills are user-invoked — this router cannot fire them for you). If two genuinely fit, name both and the deciding question above; do not pick silently. If none fit, say so — do not force a learning skill onto a non-learning request.
