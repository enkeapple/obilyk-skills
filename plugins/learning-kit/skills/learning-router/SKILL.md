# Learning Router

Six learning skills exist, all user-invoked ‚Äî nothing fires them automatically and you have to remember which fits the moment. This is the index. Tell me where you are with the material; I name the **one** skill to invoke and why, then hand off. I do not teach here ‚Äî I route.

## Route by where you are with the material

| You are‚Ä¶ | Invoke | Because |
| --- | --- | --- |
| **drowning in jargon ‚Äî it won't click** (you do NOT understand it yet) | `confusion-translator` | finds the one keystone idea and translates it into plain life, gating on comprehension |
| **fresh off learning it ‚Äî want to know if it actually stuck** | `explain-it-back` | you explain it to a 10-year-old; every gap gets stopped at, you repair it |
| **sure you've mastered it ‚Äî want that disproven** | `hidden-gaps-detector` | five deceptively simple questions + a blunt verdict on what your foundation is missing |
| **wanting to learn a concept by doing and failing, not by being told** | `learn-by-failing` | drops you in a realistic scenario, withholds the answer when you err |
| **needing to be functional FAST** (hours, not a syllabus) | `fast-track` | the highest-leverage 20%, what to ignore, one keystone rep |
| **holding a concrete outcome + a deadline** | `plan-architect` | a day-by-day plan, one 45-minute task per day, traced back to the outcome |

## Disambiguate the two close pairs

- **`confusion-translator` vs `explain-it-back`** ‚Äî the deciding factor is direction. You cannot follow it yet: use `confusion-translator` (the concept gets translated *to* you). You think you can already explain it: use `explain-it-back` (you explain it and get stopped at every gap).
- **`hidden-gaps-detector` vs `learn-by-failing`** ‚Äî the deciding factor is intent. You want your claimed mastery falsified: use `hidden-gaps-detector` (it interrogates, returns a blunt verdict). You want to build the skill through struggle: use `learn-by-failing` (it runs an error loop until the response becomes automatic).
- **`fast-track` vs `plan-architect`** ‚Äî the deciding factor is time horizon. One sitting, get usable now: use `fast-track`. Multi-day, aimed at a checkable outcome by a date: use `plan-architect`.

## How to hand off

State the matched skill by name and the one-line reason, then tell the user to invoke it. These skills are user-invoked ‚Äî this router cannot fire them. If two genuinely fit, name both and the deciding question above; do not pick silently. If none fit, say so ‚Äî do not force a learning skill onto a non-learning request.
