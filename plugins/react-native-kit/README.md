# React Native Kit

React Native domain skills. Each skill is independent and self-contained — it carries its own fixed catalog so the same pass runs the same way every time, with no shared mega-skill to wire up.

## Quickstart

- Want to know whether a screen or component is accessible — and fix what isn't? Name the target and reach for `accessibility`.
- Install it first — see [Installation](#installation).

## How it works

The skills are invoked by naming the task ("audit the accessibility of this screen", "WCAG check"). Each one names an explicit target, sweeps a fixed rule catalog, reports coverage, and — for skills that also remediate — applies fixes only after you choose a disposition.

## Installation

### Claude Code

```text
/plugin marketplace add enkeapple/obilyk-skills
/plugin install react-native-kit@obilyk-skills
```

## What's Inside

- **accessibility** — audit a React Native screen or component against WCAG 2.2 AA (expressed as concrete RN-checkable rules), report per-rule coverage, then fix the gaps after you approve them.

## Philosophy

Accessibility, performance, and the other situational concerns of a React Native app are easy to do partially and hard to do completely — a freehand pass reliably misses the same rules every time. These skills encode the full catalog so the sweep is deterministic, not dependent on what the agent happens to remember.
