# README Scaffold Example

Illustrative (a real run derives every catalog row from disk; your repo name, skills, and categories will differ). Shows the full landing-page scaffold the skill emits **when creating a fresh README** — a superpowers-style page — and how the derived skills catalog sits inside the managed marker block.

Two zones, with one hard boundary:

- **Human-owned scaffold prose** (everything OUTSIDE the markers — intro, Quickstart, How it works, Installation, Philosophy, Contributing). Emitted ONCE as `<!-- TODO -->` placeholders for a human to fill; never regenerated on a re-run, never audited. Rephrase, reorder, or drop these freely.
- **Derived catalog** (everything BETWEEN the markers — the Skills Library). Every row is derived from frontmatter per the [catalog-derivation contract](../references/catalog-derivation.md); regenerated on every re-run; audited by `auditing-readme`. Never hand-edit inside the markers.

On a re-run against an existing README, the skill replaces ONLY the managed block — the scaffold prose is left exactly as the human last left it.

````text
# <repo>

<!-- TODO: one-paragraph intro — what this repo is and who it's for. Human-authored. -->

## Quickstart

<!-- TODO: the fastest path to using these skills (install command / first invocation). Human-authored. -->

## How it works

<!-- TODO: short narrative of the model — how skills are discovered, routed, invoked. Human-authored. -->

## Installation

<!-- TODO: install steps, per platform/agent if relevant. Human-authored. -->

## Skills

<!-- skills:start -->
<!-- Generated by bootstrapping-readme — do not edit by hand; rerun the skill. -->

### apply-chain

- **[grilling](skills/apply-chain/grilling/SKILL.md)** — Use before any creative or build work to turn a fuzzy idea into a shared, concrete design.
- **[writing-specs](skills/apply-chain/writing-specs/SKILL.md)** — Use to turn an approved design or written requirements into a concrete spec.

### authoring

- **[writing-skills](skills/authoring/writing-skills/SKILL.md)** — Test-first methodology for authoring, editing, and validating skills.

### entrypoints

- **[sdd](skills/entrypoints/sdd/SKILL.md)** — Run the full spec-driven pipeline end-to-end on a feature or change.

<!-- skills:end -->

## Philosophy

<!-- TODO: the principles behind the framework. Human-authored. -->

## Contributing

<!-- TODO: how to add or change a skill, verification expectations. Human-authored. -->
````
